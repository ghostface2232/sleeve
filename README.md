# Sleeve

Paste a music link — Spotify, Apple Music, YouTube Music, anything Odesli supports — and Sleeve composes a shareable image styled around the cover art. Universal app for iOS, Android, and web, built on Expo, React Native Skia, and NativeWind v4.

## Run

```bash
npm install --ignore-scripts     # Skia's postinstall crashes on non-ASCII paths
node scripts/setup-skia-web.js   # copies canvaskit.wasm into ./public

npm run web        # Metro + CanvasKit in the browser
npm run ios        # iOS dev client  (run `npx expo prebuild` first)
npm run android    # Android dev client (run `npx expo prebuild` first)
```

Expo Go can't load Skia's native code, so iOS / Android need a custom dev build.

## Environment

```
EXPO_PUBLIC_API_PROXY=     # Cloudflare Worker proxy base URL
```

On web, Odesli and iTunes calls go through this proxy to dodge CORS — convention is `${EXPO_PUBLIC_API_PROXY}/?url=<encoded upstream>`. When empty, web falls back to direct calls (which browsers will block). Native always calls directly.

## Project layout

```
src/
  data/         link normalize, Odesli, iTunes, transport, cache
  state/        TrackProvider context
  screens/      LookupScreen — input + result preview
  composition/  Skia canvas — layers, layout, fonts, paragraphs
  app/          Expo Router routes (_layout.tsx, index.tsx, index.web.tsx)
scripts/        setup-skia-web.js
public/         canvaskit.wasm (gitignored, regenerated by postinstall)
```

## Cover resolution cascade

1. Odesli → Apple Music ID → iTunes Lookup (≥1000 px)
2. iTunes search by Odesli's title + artist, top hit kept only when title and artist closely line up
3. Best non-Apple thumbnail with URL upscaling: Tidal 1280, Pandora 1080, Deezer 1000, Yandex 1000
4. Otherwise → manual iTunes search UI

Results are cached in-memory and in AsyncStorage with a 7-day TTL, keyed by the normalized URL.
